Accumulator

The Revolutionary Compiler Language with Dynamic Multi-States

Introduction

Accumulator is a next-generation compiler system designed from the ground up to redefine how developers think about computations, variables, and programmatic states. Unlike conventional programming languages, Accumulator introduces the concept of "Thoughts" – dynamic, multi-valued entities that automatically respond to changes in their dependencies. This paradigm allows for a truly live computation model where variables and functions evolve in real-time, providing a natural, intuitive, and readable programming experience.

The language is not merely a tool for writing code; it is a system for structuring logic in a human-readable, almost narrative form. Its syntax is textual yet precise, resembling written instructions or intentions rather than cryptic symbols. This approach ensures minimal cognitive load, faster learning curves, and direct alignment with the programmer's thought process.

Core Concepts

  • Thoughts (Dumma): The fundamental data unit in Accumulator, capable of holding multiple states and values simultaneously.
  • Spark Variables: Special dynamic variables that automatically update whenever their dependent Thoughts change.
  • Chains: Ordered sequences of operations or Thoughts that propagate results through their sequence without explicit loops.
  • Portals: Safe references to Thoughts allowing shared access across multiple computations while preventing direct memory conflicts.
  • Textual Syntax: Commands and operations are expressed as readable instructions, e.g., grab a as 5 or merge x y into z.
  • Live Computation: Any change in a Spark variable or Thought triggers automatic recalculation in dependent chains, ensuring real-time correctness.
  • Parallel Multi-States: Thoughts can simultaneously exist in multiple states, allowing the system to track alternate computation paths, scenarios, or "what-if" analyses automatically.

Syntax Examples

Basic Variable Assignment

grab a as 5
grab b as 10
grab c as spark merge a b
show c   // 15

Automatic Update with Sparks

grab a as 6
show c   // 16, automatically updated due to Spark dependency

Functions as Dynamic Formulas

make circle_area with r start
    -> spark times 3.14159 r r
end

grab radius as 5
grab area as circle_area radius
show area   // 78.53975

grab radius as 6
show area   // 113.09754, dynamically updated

Conditional Thoughts

think if x greater 5 start
    show "x is greater than 5"
otherwise
    show "x is 5 or less"
end

Chains of Operations

grab x as 2
grab y as 3
grab z as spark merge x y times 2
show z   // 10
grab x as 4
show z   // 14, automatically recalculated

Unique Advantages

  • Fully dynamic variable system – Sparks and Thoughts automatically track dependencies.
  • No traditional operators or symbols – all computation expressed as human-readable commands.
  • Supports parallel computation paths through multi-state Thoughts.
  • Readable, narrative-style syntax that reduces cognitive load and improves collaboration.
  • Extensible – developers can create custom Thoughts, Sparks, Chains, and Portals as needed.
  • Live computation ensures real-time correctness without manual recalculation.

Sample Program: Dynamic Calculator

grab price as 100
grab tax_rate as 0.2
grab tax as spark times price tax_rate
grab total as spark sum price tax
show total   // 120

grab price as 150
show total   // 180, automatically updated

Design Philosophy

Accumulator is designed to mimic the natural human way of thinking about computations. Instead of focusing on low-level syntax or rigid types, the system encourages programmers to describe their intentions, relationships, and dependencies directly. Every Thought represents a mental model of a variable or operation, and Sparks provide a bridge between changing inputs and outputs. Chains allow for seamless propagation of computation without writing repetitive loops. Portals maintain safe references for distributed or shared computations.

This philosophy transforms programming from a manual instruction system into a dynamic reasoning system, where the language itself helps maintain correctness, consistency, and clarity.
